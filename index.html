kiwi

an unorthodox scripting language

kiwi is a modern, lightweight scripting language designed for simplicity, flexibility, and performance.

a quick taste of kiwi:

# a simple kiwi program to greet the world.
fn greet(name)
  println "Hello, ${name}!"
end

greet("kiwi")

Getting Started

Docker

Experiment in a Docker container.

sudo docker build -t kiwi-lang .
sudo docker run -it -v $(pwd):/workspace kiwi-lang

Linux Builds

To build, use your favorite C++ compiler.

This repo uses GCC and Make.


Fedora / RHEL

sudo dnf update
sudo dnf install gcc-c++
sudo dnf install make

Ubuntu / Debian

sudo apt update
sudo apt-get install g++
sudo apt-get install make

Then build with Make.

make

Installing on Linux

Download the repository and run sudo make install to build and run the installation script. 

git clone https://github.com/fuseraft/kiwi.git
cd kiwi
sudo make install
To manually install kiwi after it is built, run:
./bin/kiwi configure install
To uninstall kiwi, run:
kiwi configure uninstall

Windows Builds

You can find a 64-bit build (named kiwi.exe) in the latest release.

To build on Windows, you need to open a 64-bit hosted developer command prompt.

Then navigate to the source code and run build.bat.

cd path\to\kiwi
build.bat

Note: The Windows build does not support .ü•ù files (use the .kiwi extension).


Visual Studio Code Extension

You can install the extension for syntax-highlighting in VS Code.

Launch VS Code Quick Open (Ctrl+P), paste the following command, and press enter.

ext install fuseraft.kiwi-lang

Test Suite

Explore the test suite for a collection of tests.

To run the test suite, execute:

To build and run the test suite, execute:

make test

Contributions

I welcome and appreciate any and all contributions to kiwi! Here's how you can contribute:

  1. Join the Discord: The Kiwi Programming Language
  2. Fork the Repository: Fork the project to your GitHub account.
  3. Clone the Forked Repository: Clone it to your machine.
  4. Set Up Your Environment: Follow the "Getting Started" section.
  5. Make Changes: Implement your features or fixes.
  6. Test Your Changes: Ensure all tests pass.
  7. Create a Pull Request: Submit it from your fork to the main kiwi repository.

For more details, please read CONTRIBUTING.md.
